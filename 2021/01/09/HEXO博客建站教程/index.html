    打基础

    本部分适合0基础的小白，大佬绕道至文末的优化部分。

    众所周知，如果想要写点什么，可以在以下平台：

    • 第三方创作平台，如知乎，微博，简书等
    • 微信公众平台
    • 专用的记笔记软件，如印象笔记，有道云笔记
    • 自建博客

    当然，想要炫酷的还得自建博客，有多种办法自建博客，从实现原理来说，不外乎服务器、前端网页文件[酷炫特效必备文件]，服务器是用来存放我们的博客的，可以理解为是存放粮食的仓库，而前端文件就是煮成的煲仔饭，蛋炒饭等。

    本次采用Github服务器+Hexo框架+matery主题的方案，效果就是第一个链接，如果是cactus主题，效果就是第二个链接。

    为何采取这个方案呢？

    因为Github服务器不用花钱，对学生党友好，当然速度就不如阿里云，腾讯云主机啦，因为Github服务器是放在海外的，距离我们很远，访问网页自然比较慢啦。

    Hexo框架的原因是易上手，选取现成的主题也是这个原因，当然后期大家可以去Github搜索关键词hexo theme然后可以看到很多主题，爱折腾的可以自己跟着走。

    第五步：究级优化

    第四步优化主要是前端网页的展示效果的优化配置，这一步则是提高Github网页的访问速度等后端工作。

    1. CDN加速

    我们知道我们的博客有很多特效和图片，而我们的博客搭建在原在海外的GITHUB服务器，怎么加速？

    就像你买个东西，我们是不是有快递员呢，是的！用上CDN后，我们请求的图片、网页特效就不需要走这么远去访问了，只需要在距离我们的CDN提供商的服务器去访问，这里推荐jsdelivr这个CDN提供商，我们只需要把js(网页特效文件)上传到jsdelivr，然后修改配置文件相关的js引用链接就可以了，是不是很简单，对于图片，我们可以采取Github图床+PICGO，具体的话百度就可以找到教程。

    1. 添加评论系统

    评论系统强烈建议Valline，点我注册，具体教程可以参考这个，值得一说的是，有些主题例如NEXT内置了Valine的js文件，你只需要在配置文件开启并填写你的Valine帐户名和密码即可，其他情况就看看上面的这个链接就可以了，不再赘述。

    最后：常见问题

    1. hexo时显示不存在？

    如果你按照上面的安装了的话，即npm install hexo-cli g，那么可能是你没有进入主题文件夹，即通过命令行进入对应文件夹即可。

    1. hexo d报错

    首先命令行输入这个：npm install hexo-deployer-git –save，然后看自己的根目录配置文件里面是不是如下配置的：

    deploy:
    type: git
    repository: git@github.com:zhuge-ss/zhuge-ss.github.io.git
    branch: master

    如果还有问题，建议先hexo clean再hexo g -d，当然如果仍然报错，建议等几分钟再来或者重启，这是玄学：）
